Widht e Height no CSS

Normalmente os valores são definidos em porcentagem ou em pixel. widht auto pega o tamanho original de uma imagem ou preenche a div na largura da tela toda. Quando colocar como valor inherit, o tamanho é herdado da imagem anterior. É possivel também se definir tamanhos maximos e minimos max-width e min-width, nesses casos, essa definição ganha prioridade.

Algumas propriedades padrão para configurar o CSS da página

Coloca-se essa configuração para definir que o tamanho de um elemento (div) é definido pelo conteudo, padding e border, então o tamanho do padding e do border é subtraido do widht/height total definido.

* {
box-sizing: border-box;
}

Coloca-se essa configuração para redefinir a margem da pagina em 0px, dessa forma, as margins e padding são definidos exatamente pelos valores que se colocar no css.

body {
margin: 0px;
}

Formatação de texto em CSS

As propriedades abaixo podem mudar a formatação de textos em qualquer tag
color: red; - Para mudar a fonte de cor
font-size: 20px; - Para definir o tamanho da fonte
font-weight: normal ou bold; - Para colocar a fonte em negrito ou normal
font-family: helvetica, arial, serif; - Para definir a fonte a ser utilizada e fontes alternativas para o navegador utilizar em sua ausencia.
text-align: left, center, right, justify; - Para definir o alinhamento do texto
text-decoration: none, underline, overline (sublinhado acima) , line-through; - Para definir riscos e sublinhado no texto.
text-transform: uppercase ou lowercase; - Para colocar o texto todo em caixa alta ou caixa baixa
letter-spacing: 5px - Para colocar um espaçamento maior ou menor entre as letras.
word-spacing: 5px - Para aumentar ou diminuir o espaçamento entre palavras
line-height: 30px - Para aumentar ou diminuir o espaçamento entre linhas
text-shadow: 0px(direita) 2px(baixo) 3px(quão esfumaçado) red - Usado para fazer efeito de sombreamnto.

Como importar fontes de letras pela internet

Quando for utilizar uma fonte que pode não ser tão comum a todos os navegadores ou quando quiser uma fonte com estilo mais diferente, é possivel importar fontes externas de sites da internet como o googlefonts. Para inclui-la é necessário seguir esses passos:
Depois de acessar o site e copiar o link para importação é preciso colar o link no < head> assim como o link de arquivo.css. Dessa forma, fica assim:
< head>
< meta charset="UTF-8"/>
< title> titulo da pagina < /title>
< link rel="stylesheet" href="arquivo.css" />
< link rel="preconnect" href="https://fonts.googleapis.com">
< link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
< link href="https://fonts.googleapis.com/css2?family=Roboto:ital,wght@0,400;0,700;1,400&display=swap" rel="stylesheet">
< /head>
A fonte acima a ser importada foi a "roboto" regular400, italic400 e bold700 e no site as especificações para utiliza-la no CSS são as seguintes:
CSS rules to specify families
font-family: 'Roboto', sans-serif;

Caracteristicas do CSS

Herança

Ao fazer estilizações é preciso se atentar as heranças de estilizações. Como muitas tags normalmente ficam dentro de outras, pode ser que ao aplicar um efeito sobre uma determinada tag, as outras tags que estejam dentro dela ou que se repitam no html, herdem os mesmos efeitos. Dessa forma, para alterar tags especificas sem querer que as outras dentro dela ou iguais a ela fiquem com as mesmas caracteristicas, é possivel fazer o uso de id, class, ou aplicar o efeito especificamente na tag em especifico que você deseja alterar como a tag < strong id="strongred"> Em negrito e vermelho < /strong> . Nesse exemplo, foi alterada as especificações somente do texto dentro do strong e para isso foi utilizada a seguinte estilização no CSS:
.div3 #strongred {
color: red;
}

Especificidade

Quanto mais especifico voce for para identificar onde voce quer colocar o efeito, o CSS reconhece como prioridade para a aplicação dos estilos. Por exemplo:

body div.1 p {
color: red;
}
e
p {
color:blue
}

Nesse exemplo, a primeira definição ganha prioridade, por estar mais especificada, então a cor a ser aplicada vai ser o red, e não o blue.

Cascata

No Css, a ultima propriedade aplicada é a que ganha prioridade. Então a ordem de aplicação é a seguinte: Primeiro o Css aplica os efeitos do css externo, depois aplica os efeitos do css do style dentro do html, e por ultimo ele aplica os efeitos do css inline, diretamente onde esta sendo aplicado. Dessa forma, quando existir as tres formas de css e que apresentem informações distintas, a informação que vai ganhar prioridade da estilização sera o que esta no style do css inline, que é mais especifico. Dessa forma, é mais garantido que o efeito sera aplicado.

Unidades de medida absoluta no CSS

Unidades de medida absoluta são imutaveis e invariaveis, dessa forma não sofrer modificações ou interferencias. Para definir os tamanhos no CSS é possivel utilizar as seguintes formas:
Pixel = px - a forma mais utilizada, para definir tamanhos em pixel.
Pontos = pt - utilizado normalmente em tamanhos de fontes.
polegadas = in - para definir tamanho em polegadas
centimetro = cm - utilizado para definir o tamanho em centimetros
milimetros = ml - utilizado para definir tamanho em milimetros

Unidades de medida relativas a fonte

Unidades de medida relativas a fonte mudam de tamanho, baseadas em tamanhos de outros elementos existentes na pagina, são proporcionais a outra medida estabelecida anterioremente. Veja os exemplos:

< div class="divmed1">
< div class="divmed2">
Conteudo dentro da divmed2
< /div>
< /div>

CSS aplicado nessas divs: Uso da unidade de medida "em"

.divmed1 {
font-size: 40px
}
.divmed2 {
width: 10em;
height: 10em;
Color: #CCC;
}

Nesse exemplo, a primeira definição de tamanho foi o tamanho da fonte na divmed1. Dessa forma, as medidas relativas, vão utilizar como parametro essa medida para então proporcionalizar outros elementos da pagina de acordo com os 40px dessa primeira configuração. Nesse caso, 1em equivale a 40px, 2em = 80px, 3em = 120px, 10em = 400px e assim por diante. Mas é importante ressaltar que a medida utilizada como parametro é sempre a medida anterior> então os valores de 1em vão mudando ao longo da pagina, porque utilizam como parametro o valor anterior que ja foi "modificado". Exemplo:

.divmed1 {
font-size: 40px
}
.divmed2 {
width: 10em; = 400px
height: 10em;
Color: #CCC;
}
p {
font-size: 10em; = 4000px = 400*10, porque o valor estabelecido anteriormente foi equivalente a 400px
ou font-size: 1em; = 400px, porque o valor anterior utilizado foi equivalente a 400px.
}

Percebe-se que dessa forma existe um efeito cascta por toda a pagina em que uma medida altera a proporcão para definir a proxima medida.

Agora veja o CSS aplicado nas mesmas divs, com o uso da unidade de medida "rem"

Ao utilizar a unidade de medida rem, ela é utilizada como parametro em todo o site. Para defini-la como parametro, é preciso colocar ela na raiz do codigo, ou seja, é preciso aplica-la na tag do html. Ou, se não estabelecido o seu valor, por padrão 1rem equivale a 16px.

html {
fontsize: 10px
}
.divmed1 {
font-size: 4rem; = 4*10px = 40px
}
.divmed2 {
width: 40rem; = 40*10px = 400px
height: 20rem; = 20*10px = 200px
Color: #CCC;
}

Note que dessa forma, é mantida a porporção de forma fixa, baseando sempre na definição do html. Então o valor de 1rem vai ser sempre o mesmo que o valor absoluto estabelecido na tag do html. Dessa forma, garante-se uma maior esponsividade do site, uma vez que se torna totalmente proporcional e que alterando o valor do fontsize do html, voce consegue alterar os valores da pagina toda poporcionalmente, como se fosse um zoom em toda a pagina, que altera seu tamanho em conjunto.

Porcentagem, ch, ex

Porcentagem define o tamanho baseado em seu espaço livre para uso. Não é o espaço total da tela, e sim o espaço disponivel para o elemento em especifico.
ch, é utilizado em fonte monospace, em que todas as letras ocupam o mesmo tamanho. então o tamanho é definido pelo numero de caracteres dessa fonte monospace.
Ex é utilizado para definir o espaço de acrodo com o tamanho do "x" do caractere, é similar ao ch.

Funciona similiar a proporcão, só que utilizando o real espaço total da tela como parametro.

Responsividade

Responsividade diz respeito a como as configurações da pagina são adaptaveis aos diferentes formados de dispositivos e tamanhos de tela.

Mobile First

Com a ascensão do uso de disposiivos celulares, utilizava-se essa expressão para definir que deveria ser feito os codigos da pagina pensando primeiramente em como esses codigos aparentariam e performariam nos celulares, e somente depois, adaptar o codigo ao navegador. Contudo, foram identificados alguns problemas ao fazer isso, principalmente no que diz respeito as limitações que os dispositivos moveis apresentam na hora de performar os codigos da pagina. Dessa forma, hoje em dia não se faz mais isso, mas o conceito é utilizado para levar em conta a importancia de se pensar em como os os codigos da pagina podem ser adaptados para outros dispostivos também.

Layout Responsivo x Adaptativo

Layout não responsivo:
Nesse tipo de layout, as configurações do site não se adaptam ao tamanho da tela, então o mesmo layout do navegador é o que aparece em outros tipos de dispositivos ou tamanhos de tela.
Layout Responsivo:
Nesse tipo de Layout, existe uma adaptabilidade fluida, flexivel e "automatica" aos diferentes tamanhos de tela, os tamanhos não são fixos e sim flexiveis.
Layout Adaptativo:
Nesse tipo de Layout, são definidas configurações de tamanhos no site diferentes para os diferentes tamanhos de tela. Existe uma adaptabilidade de acordo com uma "faixa" de tamanho de tela. Então a responsividade é baseada em qual o tamanho da tela e qual configuração cabe nesse tamanho de tela. Não é fluido.
O ideal é misturar as formas de layout adaptativo com responsivo.

Tamanhos de telas
320 celular normal
480 celulares notes
760 tablets
960 monitores menores
1200 monitor fullhd normal
1600 monitor fullhd, 4k maior.

O ideal seria fazer 3 layouts adaptativos com responsividade para outras faixas de tamanho de tela. exemplo:

960 responsivo para até 1200/1600 760 responsivo para até 959 320 responsivo para até 759

Viewport

O viewport é a área onde seu website aparece. É a área branca da janela quando você abre o browser. O viewport sempre vai ter o tamanho da janela. Mas a forma como os elementos são renderizados vai depender do dispositivo. Para garantir que o layout da tela seja responsivo para os diferentes tipos de dispositivos e telas, é preciso colocar uma tag no head do html. veja abaixo:

< meta name="viewport" content="width=device-width, initial-scale=1.0">
ou
< meta name="viewport" content="width=device-width", uderscalable="no"> - Para que não seja permitido que o usuário dê zoom na tela.

Imagens Responsivas: TAGS picture e source

Para fazer com que as imagens da pagina mudem conforme o tamanho da tela, utiliza-se as seguintes tags que especificam quais as condições de tamanho de tela devem ser levadas em conta para carregar uma ou outra imagem, de acordo com o seu tamanho.

< picture>
< source media="(min-width: 650px)" srcset="img3.png"/>
< source media="(min-width: 450px)" srcset="img2.jpg" />
< img src="img1.png"/>
< /picture>
