Como fazer Formulários em HTML

Formulários são utilizados para que o usuário possa inserir/escolher, informações dentro de uma pagina. Para fazer formulários, basta seguir a seguinte estrutura:


1. Primeiro é utilizada a tag < form action="login.html" method="POST"(ou "GET") >

Nessa tag é inserida a pagina para onde as informações serão enviadas e onde o usuário será redirecionado. No form action é colocado o endereço da pagina para onde as informações irão, já no Method, é possivel inserir GET, em que as irformações ficam visiveis na url, ou POST, em que as informçaões não aparecem na url.


2. Depois são inseridas as tags de input, select, textarea, button.

Essas tags servem para definir que tipo de dados, informações e campos vão ser apresentados na pagina.

Tag Textarea

Essa tag é utilizada para inserir um campo de texto maior, ele é multilinha, ou seja, é possivel existir quebras de linhas. Em sua forma padrão, é possivel redimensionar a caixa de texto com o mouse ao clicar no canto inferior direito. Veja abaixo a estrutura da tag e como ela fica no navegador:



< textarea name="areadetextomultilinha"> < /textarea>




É possivel ainda, personalizar o tamanho da caixa de texto com o CSS e torná-lo fixo. Veja o exemplo:


< textarea name="areadetextomultilinha" style="width: 300px; height: 150px; border:1px dashed blue;"> < /textarea>



Tag Button

Essa tag é utilizada com a mesma finalidade do input submit, para enviar as informações do formulário. Veja sua estrutura e aparencia no navegador a seguir:



< button> Enviar < /button>



Tag Label

Essa tag é utilizada para escrever as instruções do campo a ser preenchido, e também para fazer a autoseleção do campo ao clicar em cima de sua descrição. Veja os dois exemplos a seguir:



Nesse exemplo, o label é utilizado para nomear/instruir sobre o que deve ser preenchido no campo. Veja:

< label>Nome completo:< /label>
< input type="text" name="nome">




Já nesse exemplo, note que é possivel selecionar o campo de texto ao clicar em seu nome/descrição, nesse caso é usao os atributos de "id" e "for".

< label for="nome">Nome completo:< /label>
< input type="text" name="nome" id="nome">




Outro exemplo de como o label para seleção pode ser utilizado. Note que nesse exemplo, o label é utilizado depois da tag de input, e não antes, dessa forma, o texto de seleção fica alinhado depois do campo de checkbox. Veja:

< input type="checkbox" name="termos" id="termos">
< label for="termos">declaro que li os termos de uso< /label>



Outra forma de organizar e estruturar o label dentro dos formulários e que dispensa o uso de "id" e "for" é a seguinte:

< label>
Nome completo: < br/>
< input type="text" name="nome"/>
< /label>



< label>
< input type="checkbox" name="termos"/>
Declaro que li os termos de uso
< /label>

required, minlength=0123..., maxlength=0123..., step=1,2,3..., min=123..., max=123..., min=2021-08-15, pattern, etc.

Autofocus, Placeholder e Maxlenght

Atraves do placeholder é possivel escrever dentro da caixa de texto a instrução sobre o que deve ser preenchido.


Através do autofocus é possivel deixar ativo o campo de texto automaticamente para o uruário escrever sem precisar clicar como mouse.


Através do maxlenght ou minlenght é possivel definir um tamanho maximo ou minimo de caracteres permitidos no campo.


Através do min e max, é possivel definir um valor maximo em numeros permitido no campo de texto.


Required com mensagem de aviso personalizado:


required oninvalid="setCustomValidity('Por favor, preencha este campo')" onchange="try{setCustomValidity('')}catch(e){}" />


Autocomplete="on" serve para autocompletar o campo com informações salvas pelo navegador. Para nao permitir o autopreenchimento no site basta colocar autocomplete="off"


Veja exemplos:



<form method="GET"> <label> Nome completo: <br/> <input class="inputmaior" type="text" name="nome" placeholder="Nome completo" autofocus maxlength="5" required oninvalid="setCustomValidity('Por favor, preencha este campo com o seu nome')" onchange="try{setCustomValidity('')}catch(e){}"/> </label> <br/> <label> E-mail: <br/> <input type="email" name="email" placeholder="seu e-mail" autocomplete="on" required oninvalid="setCustomValidity('Por favor, preencha este campo com o seu e-mail')" onchange="try{setCustomValidity('')}catch(e){}" /> </label> <input type="submit" value="Enviaar"/> </form>

TAG Input type hidden

Nessa tag, é possivel colocar um campo de informações oculto, não acessivel ao usuário e que não é permitido a sua alteração. Essa informação só é acessivel aos desenvolvedores do codigo, pode ser usado para marcar e identificar informações uteis no formulario, como ids, usuário, pais de origem, etc.




Note que ao clicar no botão de enviar, aparece na url as informações de país selecionadas automaticamente, e que não é possivel ver nenhum campo ou forma de modificá=la no formulario do navegador.

<form method="GET" enctype="multipart/form-data"> <input type="hidden" name="pais" value="BR"/> <input type="submit"/>

TAG Input type file

Nessa tag é possivel permitir que o usuário anexe e envie arquivos pelo formulario, como imagens, documentos, etc. Veja a estrutura da tag abaixo:


< input type="file"/> - Qualquer arquivo


< input type="file" accept="image/*"/> - Para imagens


< input type="file" accept="application/pdf"/> - Para pdf


Importante: Sempre que for utilizar essa tag para permitir o envio de arquivos, é necessário fazer uma especificação de encriptação dentro da tag < form>, para que o servidor receba corretamente o arquivo. Dessa forma, é preciso fazer o seguinte:

< form method="GET" enctype="multipart/form-data">
