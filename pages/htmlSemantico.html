Html Semântico

por Andressa Martins

No Html semântico, utilizamos tags especificas para semantica e acessibilidade. Essas tags são interpretadas pelos monitores de acessibiidade e são muito valorizadas pelo motor de busca do google, de forma que é muito importante tê-las para a otimização do SEO da pagina. Elas não possuem alterações visuais, mas podem funcionar como divs, sendo possivel inclusive a aplicação de efeitos no css.

Tag Header, Footer e Nav

Tag < header>

Indica aos motores de busca, que todo o conteúdo inserido nessa tag refere-se ao cabeçalho da página. Ex: titulo da pagina e subtitulo


Tag < nav>

Indica aos motores de busca, que dentro dessa tag está o menu principal da pagina, para facilitar a linkagem dos links das paginas do site, e tambem para a leitura pelos monitores de acessibilidade.


Tag < footer>

Conteudos de rodapé. Quem criou, termos de uso, ano do site, etc


Veja exemplos dessas tags aplicadas:

<html> <head> <title> titulo da pagina </title> <link href="estilo.css" rel="stylesheet"/> </head> <body> <header> <h1> titulo da pagina </h1> <p> subtitulo da pagina </p> </header> <nav> <a href="" > link 1 </a> <a href="" > link 2 </a> <a href="" > link 3 </a> </nav> <footer> <p>Rodapé da pagina</p> </footer> </body> </html>

Tag Main, Section, Article, Aside e Time

< main> < /main>

Essa tag é utilizada para definir o conteudo principal da pagina.


< section> < /section>

Esta tag separa os assuntos diferentes sobre o conteudo da pagina, não os misturando entre si.


< article> < /article>

Assunto principal da pagina, uma noticia, materia, artigo, etc. Importante para SEO.


< aside> < /aside>

Usa-se essa tag para alocar os conteudos secundário da pagina


< time> < /time>

Nessa tag é possivel especificar datas para escaneamento do codigo da pagina, não é visivel ao navegador.

Veja exemplos dessas tags aplicadas:

<html> <head> <title> titulo da pagina < /title> <link href="estilo.css" rel="stylesheet"/> </head> <body> <header> <h1> titulo da pagina </h1> <p> subtitulo da pagina </p> </header> <nav> <a href="" > link 1 </a> <a href="" > link 2 </a> <a href="" > link 3 </a> </nav> <main> <section> </section> <article> </article> <aside> </aside> <time datetime="1997-01´13"> Data do meu animersário </time> </main> </body> </html>

Considerações sobre a tag main


* só é possivel ter uma por página home.
*A tag Main só pode ser " filho" ou "pai" de section.
*A tag Main nunca será "filho" de aside, footer, nav, header e article
A tag MAIN tem uma única função semântica: definir o conteúdo principal da página ou da aplicação. Ele representa o conteúdo mais importante da página, que está diretamente relacionado ao tópico central do documento ou a funcionalidade principal de uma aplicação.

Tag address

< address> < /address>

Nessa tag se coloca informações de endereço e contato no rodapé


Veja o exemplo a seguir

<footer> <p> os direitos reservados... </p> <address> <p> Site feito por mim </p> <p> para contato: suporte@suporte.com </p> <p> Av. Consolação 1200 - Higienópolis, São Paulo.</p> </address> </footer>

Tag Figure

< figure> < /figure>

Essa tag é utilizada em Semantica para definir que uma imagem faz parte do contexto do texto, quando se quer inserir uma legenda para a imagem.


Veja o exemplo a seguir


<figure> <img src="../assets/images/flower.jpg"/> <figcaption>Imagem de uma flor</figcaption> </figure>

Para a estilização nesse caso, foram usadas as seguintes propriedades:


figure {
padding: 5px;
margin: 0px;
border: 1px solid black;
text-align: center;
}

figcaption {
background-color: grey;
}

Tags de citações

< cite> < /cite>

O elemento < cite> representa uma referência a um trabalho artístico. Deve incluir o título do trabalho ou uma URL de referência, que pode ser em uma forma abreviada de acordo com as convenções usadas para a adição dos metadados de citação.< /cite>


< blockquote> < /blockquote>

O Elemento < blockquote> indica que o texto incluído é uma longa citação. Normalmente, este é processado visualmente pelo recuo.< /blockquote>


< q> < /q>

O elemento < q> indica que o texto em anexo é uma cotação curta em linha.< /q> A maioria dos navegadores modernos implementam isso cercando o texto entre aspas. Este elemento é destinado a citações curtas que não exigem quebras de parágrafo; para citações longas, use o elemento < blockquote>.
